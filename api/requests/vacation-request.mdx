## Endpoint

```
POST /api/solicitudes/crearSolicitudVacaciones
```

This endpoint creates vacation requests with automatic validation of available vacation days. The system checks the employee's balance before creating the request.

## Request Body

<ParamField body="fecha_inicio" type="string" required>
  Vacation start date in ISO format or YYYY-MM-DD
  
  Example: `"2026-05-01"` or `"2026-05-01T00:00:00"`
</ParamField>

<ParamField body="fecha_fin" type="string" required>
  Vacation end date in ISO format or YYYY-MM-DD
  
  Must be equal to or after `fecha_inicio`
</ParamField>

<ParamField body="descripcion_solicitud" type="string">
  Optional description for the vacation request
  
  Example: `"Summer vacation"`
</ParamField>

<ParamField body="documento_url" type="string">
  Optional URL to supporting document
</ParamField>

## Response

<ResponseField name="message" type="string">
  Success message: "Solicitud de vacaciones creada exitosamente"
</ResponseField>

<ResponseField name="solicitud" type="object">
  <Expandable title="properties">
    <ResponseField name="id_solicitud" type="number">
      Unique identifier for the vacation request
    </ResponseField>
    <ResponseField name="id_empleado" type="number">
      Employee who created the request
    </ResponseField>
    <ResponseField name="id_tipo_solicitud" type="number">
      Always `1` for vacation requests
    </ResponseField>
    <ResponseField name="id_estado_solicitud" type="number">
      Initial state (always `1` for Pending)
    </ResponseField>
    <ResponseField name="fecha_inicio" type="string">
      Vacation start date (YYYY-MM-DD HH:mm:ss)
    </ResponseField>
    <ResponseField name="fecha_fin" type="string">
      Vacation end date (YYYY-MM-DD HH:mm:ss)
    </ResponseField>
    <ResponseField name="dias_solicitados" type="number">
      Number of vacation days requested
    </ResponseField>
    <ResponseField name="descripcion_solicitud" type="string">
      Request description
    </ResponseField>
  </Expandable>
</ResponseField>

## Validation Rules

<CardGroup cols={2}>
  <Card title="Date Validation" icon="calendar-check">
    - Start date must be before or equal to end date
    - Dates cannot be in the past
    - Date format must be valid
  </Card>
  <Card title="Balance Validation" icon="scale-balanced">
    - System checks employee's available vacation days
    - Request is rejected if insufficient days available
    - Balance is only deducted after approval
  </Card>
</CardGroup>

## Code Examples

<CodeGroup>
```javascript JavaScript
const token = 'your_jwt_token';

const vacationData = {
  fecha_inicio: '2026-07-01',
  fecha_fin: '2026-07-14',
  descripcion_solicitud: 'Summer vacation'
```

```python Python
import requests
import json

token = 'your_jwt_token'
headers = {
    'Authorization': f'Bearer {token}',
    'Content-Type': 'application/json'
}

vacation_data = {
    'fecha_inicio': '2026-07-01',
    'fecha_fin': '2026-07-14',
    'descripcion_solicitud': 'Summer vacation'

    '/api/solicitudes/crearSolicitudVacaciones',
    headers=headers,

)
data = response.json()
print(data)
```

## Business Logic Flow

<Steps>
  <Step title="Retrieve Employee Balance">
    System loads the authenticated employee's vacation balance from the database
  </Step>
  <Step title="Validate Dates">
    Checks that dates are valid and in correct order using `validarFechasSolicitud()`
  </Step>
  <Step title="Calculate Days">
    Computes the number of vacation days using `calcularDiasSolicitados()` (excludes weekends)
  </Step>
  <Step title="Check Balance">
    Compares requested days against available balance
  </Step>
  <Step title="Create Request">
    If validation passes, creates the request in pending state
  </Step>
  <Step title="Balance Deduction">
    Days are only deducted from balance when the request is approved (not at creation time)
  </Step>
</Steps>
